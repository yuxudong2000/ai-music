# AI Music 工具 - 需求描述文档

> **版本**：v0.7  
> **创建日期**：2026-03-16  
> **状态**：草稿

---

## 1. 项目概述

本项目旨在开发一款命令行音乐处理工具，基于 AI 技术，提供四项核心能力：

1. **声音学习（Voice Learning）**：通过提供一个或多个参考音频，让系统学习并保存目标人声的音色模型，供后续反复使用。
2. **声音替换（Voice Replacement）**：在不改变歌曲伴奏、音效及其他音频属性的前提下，将歌曲中的人声替换为已学习的目标人声音色。
3. **歌词生成（Lyrics Generation）**：用户提供原歌词文本作为结构约束来源，由 AI 自动生成符合约束的新歌词文件，用户可在此基础上进一步手动编辑完善。
4. **歌词替换与合成（Lyrics Replacement & Re-synthesis）**：在保持原有旋律和人声音色不变的前提下，将歌曲中的歌词替换为指定歌词文件中的新歌词，并重新合成完整歌曲音频。

**目标用户**：个人娱乐用户、独立音乐创作者  
**交付形式**：命令行工具（CLI）  
**第一期语言支持**：中文（设计需考虑未来扩展英文等多语言）

## 2. 功能需求

### 2.1 功能一：声音学习（Voice Learning）

#### 2.1.1 功能描述

用户提供一个或多个包含目标人声的参考音频文件，系统对这些音频进行学习，提取并保存音色特征模型，并为其命名。保存后的声音模型可在功能二（声音替换）中反复使用，无需每次重新提供参考音频。

#### 2.1.2 声音模型来源

| 来源 | 描述 |
|---|---|
| **用户自定义学习** | 用户提供一个或多个目标人声的参考音频（如目标歌手的录音片段），系统学习并保存为命名的声音模型 |
| **系统预设** | 系统内置若干预设声音模型，随工具一起分发，无需用户手动学习注册 |

无论来源如何，所有声音模型在使用时均通过统一的**声音名称（Voice Name）**引用。

#### 2.1.3 输入规格

| 项目 | 规格 |
|---|---|
| 参考音频格式 | MP3（后续可扩展） |
| 参考音频数量 | 支持传入 1 个或多个文件（多个文件有助于提升模型质量） |
| 参考音频时长建议 | 每段建议不低于 10 秒，总时长建议 1~5 分钟 |
| 声音名称 | 用户自定义，仅允许字母、数字、下划线、中文，唯一标识 |

#### 2.1.4 CLI 接口示例（示意，非最终设计）

```bash
# 学习单个参考音频并保存为声音模型
ai-music voice learn \
  --name "jay_chou" \
  --ref ref_audio.mp3

# 学习多个参考音频（提升模型质量）
ai-music voice learn \
  --name "jay_chou" \
  --ref ref1.mp3 ref2.mp3 ref3.mp3

# 向已有声音模型追加新的参考音频（更新/增强模型）
ai-music voice update \
  --name "jay_chou" \
  --ref new_ref1.mp3 new_ref2.mp3

# 查看所有已保存的声音模型（含用户自定义与系统预设）
ai-music voice list

# 删除用户自定义的声音模型
ai-music voice remove --name "jay_chou"
```

### 2.2 功能二：声音替换（Voice Replacement）

#### 2.2.1 功能描述

用户输入一首歌曲音频，并指定一个已通过功能一学习保存的声音模型名称（或系统预设声音），工具自动将歌曲中的人声音色替换为目标声音，同时保持伴奏、混响、音效等所有其他音频元素完全不变。

#### 2.2.2 目标声音来源

| 来源 | 描述 |
|---|---|
| **用户自定义模型** | 通过功能一 `voice learn` 学习并保存的声音模型，通过名称引用 |
| **系统预设模型** | 系统内置的预设声音模型，通过名称引用，与自定义模型接口完全一致 |

#### 2.2.3 处理约束

- 伴奏、背景音乐、混响、音效等非人声部分**完全保留，不做任何修改**
- 替换后的人声需与原曲节奏、旋律时间轴保持对齐
- 第一期仅支持**中文歌曲**，系统架构需预留多语言（英文等）扩展接口

#### 2.2.4 输入输出规格

| 项目 | 规格 |
|---|---|
| 输入音频格式 | MP3（后续可扩展） |
| 输出音频格式 | MP3 |
| 音频时长限制 | 最长 **10 分钟** |

#### 2.2.5 CLI 接口示例（示意，非最终设计）

```bash
# 使用用户自定义学习的声音替换
ai-music voice-replace \
  --input song.mp3 \
  --voice "jay_chou" \
  --output output.mp3

# 使用系统预设声音替换
ai-music voice-replace \
  --input song.mp3 \
  --voice "preset_male_01" \
  --output output.mp3
```

### 2.3 功能三：歌词生成（Lyrics Generation）

#### 2.3.1 功能描述

用户**直接提供**歌词文本，并提供创作主题/风格等提示词，由 AI 依据歌词结构约束生成新歌词，并输出为可编辑的文本文件。用户可在此基础上对歌词进行手动修改和完善。

#### 2.3.2 歌词结构来源

系统支持以下方式提供歌词结构约束：

| 来源方式 | 描述 | 适用场景 |
|---|---|---|
| **直接输入歌词文本** | 用户提供原歌词文本文件（TXT 格式），系统从中提取段落数、句数、字数等结构约束 | 已有歌词文本、无需处理音频时 |


#### 2.3.3 歌词生成方式

| 方式 | 描述 |
|---|---|
| **AI 自动生成** | 用户提供主题、风格、关键词等提示词，由 AI 依据结构约束自动生成新歌词 |
| **结构模板导出** | 系统仅导出原歌词的结构模板（空歌词框架），供用户完全手动填写 |

#### 2.3.4 歌词结构约束

AI 生成歌词时，系统从歌词来源中提取以下约束并传递给 AI；若用户导出结构模板后手动填写的歌词不满足段落数量或句数约束，系统应在执行前给出警告，列出具体差异，并提示用户确认是否继续。

| 约束项 | 要求 | 说明 |
|---|---|---|
| **段落数量** | 必须与原歌词相同 | 每个段落对应一段旋律，数量变化会导致旋律错位 |
| **每段句数** | 必须与原歌词相同 | 每句对应一条旋律线，句数变化会导致旋律空白或截断 |
| **每句字数/音节数** | 应尽量与原歌词保持一致 | 字数差异过大会导致发音被拉伸或压缩，影响合成质量 |
| **押韵/韵脚** | 可选，不强制要求 | 不影响技术合成，但影响歌词的音乐性和听感 |
| **语义/主题** | 不限制 | 完全由用户自由指定 |

#### 2.3.5 输出歌词文件格式

生成的歌词文件为 **TXT 格式（UTF-8 编码）**，每行一句歌词，段落之间以空行分隔，并保留结构注释（如 `# [第一段]`），方便用户对照编辑：

```
# [第一段]
第一句歌词
第二句歌词
第三句歌词

# [第二段]
第四句歌词
第五句歌词
```

#### 2.3.6 CLI 接口示例（示意，非最终设计）

```bash
# 方式一：直接输入原歌词文本，AI 生成新歌词
ai-music lyrics generate \
  --lyrics original_lyrics.txt \
  --prompt "写一首关于夏天海边的歌词" \
  --output new_lyrics.txt

# 方式二：从歌词文本导出结构模板（空框架，供手动填写）
ai-music lyrics template \
  --lyrics original_lyrics.txt \
  --output lyrics_template.txt
```

### 2.4 功能四：歌词替换与合成（Lyrics Replacement & Re-synthesis）

#### 2.4.1 功能描述

用户输入一首歌曲音频和一个歌词文件（可来自功能三生成后编辑的结果，也可完全手动创作），工具自动将原歌词替换为新歌词，并基于原有旋律和原歌手音色，重新合成包含新歌词的完整歌曲音频。

#### 2.4.2 歌词文件来源

| 来源 | 描述 |
|---|---|
| **功能三生成后编辑** | 使用 `lyrics generate` 或 `lyrics template` 命令生成文件，在文本编辑器中手动调整后使用 |
| **完全手动创作** | 用户完全自行编写歌词文件，按照规定格式保存后传入 |

#### 2.4.3 处理约束

- 原曲旋律、伴奏、原歌手音色**完全保留，不做任何修改**
- AI 合成新歌词时，需基于原旋律的时间轴进行对齐
- 第一期仅支持**中文歌曲**，系统架构需预留多语言（英文等）扩展接口

#### 2.4.4 输入输出规格

| 项目 | 规格 |
|---|---|
| 输入音频格式 | MP3（后续可扩展） |
| 输入歌词格式 | TXT 文本文件（UTF-8 编码） |
| 输出音频格式 | MP3 |
| 音频时长限制 | 最长 **10 分钟** |

#### 2.4.5 CLI 接口示例（示意，非最终设计）

```bash
# 使用歌词文件替换并重新合成
ai-music lyrics-replace \
  --input song.mp3 \
  --lyrics new_lyrics.txt \
  --output output.mp3
```

### 2.5 功能组合使用

四项核心能力作为**独立命令**提供，用户可灵活组合。典型工作流如下：

**工作流 A：只换声音**
```
① voice learn（声音学习，首次使用时执行一次）→ ② voice-replace（声音替换）
```

**工作流 B：只换歌词**
```
① lyrics generate（AI 生成歌词）→ 手动编辑歌词文件 → ② lyrics-replace（歌词替换合成）
```

**工作流 C：换声音 + 换歌词**
```
① voice learn（声音学习）
② lyrics generate（生成歌词）→ 手动编辑
③ lyrics-replace（歌词替换合成）
④ voice-replace（声音替换）
```

> **注意**：工作流 C 中声音学习（①）只需执行一次，后续可跳过。建议先完成歌词替换合成（③），再进行声音替换（④），以保证处理链路清晰、中间产物可复用。

## 3. 非功能需求

### 3.1 语言与国际化

| 阶段 | 支持语言 |
|---|---|
| 第一期（当前） | 中文 |
| 未来扩展 | 英文、日文等（架构需预留扩展点） |

系统设计时需将**语言处理模块**（语音识别、TTS、歌词对齐）设计为可替换/可插拔的组件，以便后续支持多语言时无需大规模重构。

### 3.2 性能

- 单首歌曲处理时长：**不超过歌曲本身时长的 3 倍**（如 3 分钟歌曲，处理时间不超过 9 分钟）
- 支持本地 GPU 加速（有 GPU 时自动启用）

### 3.3 易用性

- CLI 命令需有清晰的帮助文档（`--help`）
- 处理过程需有进度提示（如进度条或阶段日志）
- 错误信息需友好、可读，并给出可能的解决建议

### 3.4 版权与合规

- 暂不做版权限制处理（后续可根据需要补充）

## 4. 系统边界与假设

1. 本工具作为**本地命令行工具**运行，不涉及云端部署（第一期）
2. 输入音频假设为**已经混音完成的歌曲**（含人声和伴奏），工具需具备人声与伴奏分离的能力（作为内部处理步骤）
3. 系统假设输入音频的**人声部分为单一歌手**（暂不支持多人合唱的精确分离）
4. AI 生成歌词时，结构约束（段落数、句数、字数）可从用户提供的原歌词文本中提取

## 5. 技术方案设计决策项

> 以下问题不属于需求范畴，将在下一阶段**技术方案设计**中作为关键决策项逐一澄清和确定。

| 编号 | 决策项 | 所属模块 |
|---|---|---|
| T-01 | 系统预设人声的数量、风格如何定义和维护？ | 声音学习 |
| T-02 | 声音学习/训练的预期耗时目标（如：1 分钟参考音频，训练时间不超过 X 分钟）？ | 声音学习 |
| T-03 | 声音模型文件的本地存储路径规范（如 `~/.ai-music/voices/`）？是否支持跨机器导入/导出？ | 声音学习 |
| T-04 | 人声与伴奏分离的技术方案选型（如 Demucs、Spleeter 等）？ | 声音替换 |
| T-05 | 人声音色转换的技术方案选型（如 RVC、so-vits-svc 等）？ | 声音替换 |
| T-06 | 歌词替换与重合成的技术方案选型（TTS 引擎 + 旋律对齐方案）？ | 歌词替换与合成 |
| T-07 | AI 歌词生成调用哪个大模型/服务（如 OpenAI、Claude、本地模型等）？ | 歌词生成 |
| T-08 | 工具的目标运行环境要求（操作系统、最低 GPU/CPU 规格）？ | 全局 |

## 6. 术语表

| 术语 | 说明 |
|---|---|
| Voice Learning | 声音学习，通过提供参考音频让系统学习目标人声的音色特征，并以命名方式保存为可复用的声音模型 |
| Voice Replacement | 声音替换，利用已学习的声音模型，将歌曲中的人声音色进行替换，不改变伴奏与其他音频元素 |
| Voice Model | 声音模型，由声音注册过程生成的，描述特定人声音色特征的数据文件 |
| Lyrics Generation | 歌词生成，基于原歌曲的结构约束（段落数、句数、字数），由 AI 自动创作新歌词并输出为可编辑文本文件 |
| Lyrics Template | 歌词结构模板，仅包含段落和句数框架的空白歌词文件，供用户手动填写 |
| TTS | Text-to-Speech，文字转语音技术 |
| Vocal Separation | 人声分离，从混音歌曲中分离出纯人声和伴奏两个音轨 |
| Re-synthesis | 重合成，将新歌词按照原旋律时间轴重新生成演唱音频 |
| Lyrics Alignment | 歌词对齐，将歌词文本与音频时间轴进行匹配 |
